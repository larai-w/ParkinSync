# 紙の介護ログを、無理なく見える化する ParkinSync

介護の現場では、毎日の体調、服薬、排便、気分、転倒リスクのような情報が紙に残されることが多くあります。

紙は悪い仕組みではありません。高齢の家族や介護者にとっては、スマートフォンや複雑なアプリよりも、紙のほうが速く、確実で、続けやすいことがあります。

ただ、そのままだと記録は「その日のメモ」で止まりがちです。数週間、数か月の変化を振り返ったり、服薬の時間、室内温度、天気、体調の変化を並べて見ることは簡単ではありません。

ParkinSync は、この紙のよさを残しながら、必要な部分だけをクラウドにつなぐために作った仕組みです。

## 紙をやめるのではなく、紙を活かす

最初からすべてをアプリ化すると、入力する人の負担が増えます。

ParkinSync では、介護者が書き慣れた紙の記録を起点にします。記録はあとから確認し、人の目で整えてからクラウドへ取り込みます。

この設計にした理由は単純です。介護記録では、間違ったデータを大量に集めるより、少なくても確実なデータを残すほうが大切だからです。

## 見えるようにしたかったこと

ParkinSync で見たかったのは、単なる「記録のデジタル化」ではありません。

たとえば、次のような変化です。

- 服薬の時間が日によってどれくらいずれるか
- 体調が崩れやすい時間帯があるか
- 室温や天気の変化と体調に関係がありそうか
- 介護メモの内容をあとから振り返りやすい形にできるか

これらは、1日分だけ見ても分かりません。日々の小さな記録を同じ形式で積み重ねることで、初めて見えるものがあります。

## 介護者の負担を増やさないために

ParkinSync では、介護者に新しい操作をたくさん覚えてもらうことを目標にしていません。

大事にしたのは、今ある記録の流れを大きく壊さないことです。

紙に書く。確認する。必要な部分だけをデータ化する。あとはクラウド側で集計し、日ごとの温度や天気の情報と一緒に整理する。

この流れなら、現場の入力負担を増やしすぎずに、あとから振り返れるデータを作れます。

## 技術は裏側に置く

裏側では AWS Lambda、S3、Google Sheets、IoT センサー、天気 API を使っています。

ただ、使う人にとって大切なのは、どのサービスを使っているかではありません。

大切なのは、毎日の記録が消えずに残り、あとから見返せて、家族や支援者が状況を理解しやすくなることです。

ParkinSync は、介護の記録を「その場限りのメモ」から「あとで役に立つ情報」に変えるための試みです。

## これから

今後は、匿名化したサンプルデータと公開用の説明資料を整えながら、誰が見ても仕組みを理解できる形にしていきます。

医療判断を自動化するものではなく、家族や介護者が日々の変化を振り返るための補助ツールとして、慎重に育てていく予定です。
